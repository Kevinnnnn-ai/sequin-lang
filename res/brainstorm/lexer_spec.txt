│ TOKEN INVENTORY                                                    │   │ SEQUIN LEXER — TOKEN SPECIFICATION                                    │
│ - KEYWORD ............... exactly I, M, E, R, A (reserved words)   │   │ Closed during lexer review discussion, 2026-07-18.                    │
│ - IDENTIFIER ............ commands, e.g. INSTRUMENT_POWER_STATE    │   │ Inspired by NASA-AMMOS Aerie's SeqN sequencing notation.              │
│ - FLAG .................. &NAME, ampersand glued to the name       │   └───────────────────────────────────────────────────────────────────────┘
│ - LITERAL_STRING ........ "..." single line, no escapes            │   │ WORDS (KEYWORDS, COMMANDS, FLAG NAMES)                               │ 
│ - LITERAL_NUMBER ........ integers and floats, optional sign       │   │ - Charset: letters, digits, underscore, dash.                        │ 
│ - LITERAL_TIMESTAMP ..... epoch, YYYY...-DDDT##:##:##.###          │   │ - Must START with a letter or underscore.                            │ 
│ - LITERAL_DURATION ...... relative time, H...:##:##.###            │   │   - A leading digit or sign always means number/time,                │ 
│ - DELIMITER_* ........... ( ) { } , :                              │   │     so "3D_SCAN" and "-X" are illegal command names.                 │ 
│ - NEWLINE ............... statement terminator                     │   │ - Scan the whole word greedily, THEN check the keyword set:          │ 
│ - END_OF_FILE ........... emitted exactly once, at the end         │   │   - word is exactly I/M/E/R/A -> KEYWORD (case-sensitive)            │ 
│ - (COMMENT, WHITESPACE .. no longer emitted; drop from enum?)      │   │   - otherwise -> IDENTIFIER                                          │ 
└────────────────────────────────────────────────────────────────────┘   │   - "E-FOO" is therefore one IDENTIFIER, never keyword E + junk.     │ 
│ DISPATCH (what the first character of a token decides)             │   │ - FLAG: '&' immediately followed by a word; one token.               │ 
│ - space, tab ......... skip (no token), column++                   │   │   ("& GE" with a space is illegal — the wishlist comment is a typo.) │ 
│ - newline ............ emit NEWLINE, line++, column reset          │   └──────────────────────────────────────────────────────────────────────┘ 
│ - # .................. consume to end of line, emit nothing        │   │ STRINGS                                                              │ 
│ - " .................. string scan                                 │   │ - Delimited by ", must open and close on the same line.              │ 
│ - & .................. word scan -> FLAG                           │   │ - No escape mechanism. " can never appear inside a string.           │ 
│ - letter, _ .......... word scan -> KEYWORD or IDENTIFIER          │   │ - ' is an ordinary character, inside or outside strings.             │ 
│ - digit, +, - ........ digit run, then peek one character:         │   │ - # inside a string is literal, not a comment.                       │ 
│     '-' -> timestamp   ':' -> duration                             │   │ - Newline or end-of-file before the closing " is an error.           │ 
│     '.' -> float       else -> integer                             │   │ - Because value == source bytes, tokens can stay                     │ 
│     (after an explicit sign, only integer/float are legal)         │   │   pointer+length views into the source buffer.                       │ 
│     Once committed, a malformed tail is a LEX ERROR:               │   └──────────────────────────────────────────────────────────────────────┘ 
│     "120-XYZ" errors, it is not number-then-command.               │   │ NUMBERS                                                              │ 
│ - ( ) { } , : ........ that delimiter token                        │   │ - Integers and floats only.                                          │ 
│ - anything else ...... lex error                                   │   │ - Optional explicit + or - (positive is silent by default).          │ 
└────────────────────────────────────────────────────────────────────┘   │ - Floats require digits on BOTH sides of the dot                     │ 
│ ERRORS (every error should carry line + column)                    │   │   ("1." and ".5" are illegal).                                       │ 
│ - Unterminated / multi-line string.                                │   └──────────────────────────────────────────────────────────────────────┘ 
│ - Malformed time after committing to a time shape.                 │   │ TIMES                                                                │ 
│ - Sign not followed by a digit.                                    │   │ - Strict shapes, validated in the lexer; only the FIRST field        │ 
│ - '&' not followed by a word-start character.                      │   │   is variable width. Times are unsigned.                             │ 
│ - Character matching no rule above.                                │   │ - Epoch (after E or A):   YYYY...-DDDT##:##:##.###                   │ 
└────────────────────────────────────────────────────────────────────┘   │     year: 1+ digits, DDD: exactly 3, hh/mm/ss: exactly 2,            │ 
│ CONVENTIONS / ASSUMPTIONS                                          │   │     mmm: exactly 3, .mmm mandatory                                   │ 
│ - Lines and columns are 1-based (match editors).                   │   │ - Duration (after R):     H...:##:##.###                             │ 
│ - Keywords are case-sensitive ("i" is a command).                  │   │     hours: 1+ digits, mm/ss: exactly 2, mmm: exactly 3               │ 
│ - Exactly one END_OF_FILE token, stamped with the final position.  │   └──────────────────────────────────────────────────────────────────────┘ 
└────────────────────────────────────────────────────────────────────┘
